# 給食献立ビューアー

PDFから読み取った献立を、日付ごとのカード形式で見やすく表示するWebアプリです。

## ローカル起動

```bash
cd '/Users/mba/Documents/New project'
python3 -m http.server 8000
```

ブラウザ:
- http://127.0.0.1:8000

## 追加した機能

- 月タブ切り替え（`data/menu-data.json` 内の複数月を自動表示）
- 検索/曜日フィルタ
- - 日ごとの評価入力
- 共有集計表示（Supabase設定時）
- スマホ最適化（1カラム表示、44pxタップ領域、横スクロール月タブ）
- PDF更新の半自動化スクリプト

## 共有集計の設定（Supabase）

`supabase-config.js` に値を入れると、全ユーザーの評価が集計表示されます。

```js
window.APP_CONFIG = {
  supabaseUrl: "https://xxxx.supabase.co",
  supabaseAnonKey: "YOUR_ANON_KEY",
};
```

Supabase SQL Editor で以下を実行:

```sql
create table if not exists public.meal_ratings (
  id bigint generated by default as identity primary key,
  menu_id text not null,
  day int not null,
  client_id text not null,
  vote text not null check (vote in ('good', 'bad')),
  created_at timestamptz not null default now(),
  unique (menu_id, day, client_id)
);

alter table public.meal_ratings enable row level security;

create policy "read meal ratings"
on public.meal_ratings
for select
using (true);

create policy "insert meal ratings"
on public.meal_ratings
for insert
with check (true);

create policy "update meal ratings"
on public.meal_ratings
for update
using (true)
with check (true);

create policy "delete meal ratings"
on public.meal_ratings
for delete
using (true);
```

注記:
- Supabase未設定でもアプリは動きます（この端末のローカル保存のみ）。
- 共有集計ON時は、各ブラウザごとに1端末1票として集計します。

## PDF更新の手順（半自動）

1. 新しいPDFを配置
2. 変換スクリプトを実行

```bash
cd '/Users/mba/Documents/New project'
python3 tools/update_from_pdf.py \
  --pdf '/path/to/new-menu.pdf' \
  --month 2025-03 \
  --course A \
  --title '3月 給食献立ビューアー（A献立）' \
  --pdf-link 'menu-source-2025-03-a.pdf' \
  --out data/menu-data.json
```

3. 生成された `data/menu-data.json` の `dishes`/`staple` を目視で微調整

注記:
- 栄養値・日付は高精度で拾います。
- 料理名と主食はPDFのレイアウト依存が大きいので、最終確認が必要です。
